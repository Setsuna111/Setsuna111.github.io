2021美赛B题O奖论文阅读笔记

发表于 2022-02-06 更新于 2022-02-25 分类于 Math Modeling
2021年美赛B题O奖论文集阅读笔记

1. 题目解读

1.1 题目要做的事 - background and description

  1. SSA无人机：监控消防员的数据 & 火场情况（肯定会有这个吧，不然高清热像仪和遥测传感器干啥）
  2. EOC：接受无人机的信号，决策，发出指令控制工作人员
  3. 通信怎么实现：双向无线电通信（手持式），部署人员携带（部署人员属于消防员吗？
    给出的指标：功率5W，平原5km城市2km
  4. 中继器：用于扩展无线电传播范围
    对中继器的携带：悬停无人机，20km范围，成本$10000，功能参数见表格
    有一个注：内置电池充电的时候可以更换视频/遥测的辅助电池，是说他们互不影响吗

总结一下：SSA无人机和消防员在一起监控指标，需要通过中继器无人机与EOC通信

1.2 Requirements

  1. 建立模型计算SSA无人机和无线电中继器无人机的最佳数量和组合。
    需考虑：安全性、经济型 观测需求、地形 火灾事件大小和频率（可以由已有数据估算，也便于第二个要求实现）
  2. 模型适应未来十年极端火灾事件发生的能力（就是重新算一遍吧），可以根据一些潜在问题合理推断哪里易发生火灾
  3. 【新】模型，为了应对不同地形上不同规模的火灾，优化悬停中继器无人机的位置
    给了海拔的提示图

1.3 一些想法

  1. 主模型应该比较庞大，设计的参数和因素比较多，但第二问就是在主模型基础上做了
  2. 第三问应该是对模型进行修改？那这样第一问是否需要考虑地形。以及第一问考虑最佳组合和数量也应该给出地形的考虑吧

    很自然的思路，就和要对每一个网格计算属性一样，先把网格分类
    三种：BG（屏障区域 即山脉 无人机无法飞过的（这个思路挺新，相当于是添加了飞行距离的障碍，估计是查看参考文献得到的，但这篇文章给的参考文献不是很清晰。。）） RG（漫游区域 EOC的位置） WG（着火区域 即SSA要飞到的地方 这也验证了一开始的理解，SSA是紧跟消防员到达着火区域的）

    他这里设置浅蓝色为可能的漫游无人机存在的区域，后续定义每个着火网格的频率后计算了每一个漫游无人机格子对应的指标分数（见14）

  13. 表格代表野火发生的次数，容易想到的思路就是统计一个网格里野火发生的平均频率作为其代表值
    注意统计过程使用比较math的方法写了出来，这篇文章在这里做的挺好的

  14. 定义的SC分数：很直观，人为定义的罢了 分数越大代表更应该布置（直观来看就可以，越大代表次数越多，距离越近）
    注意：直接通过分析得到了应该选择前4个排名的EOC，并没有将数量作为一个优化指标！！
    进一步考虑：不只考虑单独分数的排序，图8的流程图
    具体什么意思：当选定的4个EOC存在在3格距离之内的格子时要进行修正（好家伙，这个模型挺细的，在常规思路上做了很多细节化的考虑）
    注意 这里只是EOC的设定，并没有规划好无人机怎样发送，两种无人机的部署应该是一个动态的过程吧，但需要根据成本给出一个最优的数量

  15. 多目标优化：
    一些符号表述：意思就是两种无人机都是在某个EOC发送过去的
    两个目标函数：灭火最短时间（无人机的部署时间，很math的表述 最小化最大时间！） 和 最小化无人机数量（表示成0 1变量）
    统合两个目标的方法：加权 很直接的一种表述（注意文章还提到了另一种-分阶段优化算法）
    给出的限制：工作时间限制，有一个问题他没有考虑往返距离要*2 hhh 地形限制&高度限制（麻了 文章没说 这太离谱了。。。。不说明直接给仿真结果，鬼知道考虑没考虑

  16. 计算：使用python仿真
    （使用的什么算法啊。。。。。文章竟然说仿真过程比较冗杂就没提
    提到直接考虑最大距离的情况作为优化目标（也是 既然求最小值那就考虑最坏的情况，不用考虑频次了，也就是在定EOC时已经利用了频次，之后以确保覆盖所有着火情况作为固定约束），但更科学的话。。。考虑频次算期望不就是了，。，，
    根据充电时间直接*2？？？？？？？麻了 十分不理解这个仿真怎么做的

  17. 使用泊松模型预测极端火灾的次数
    怎么想到的：有文献直接使用泊松模型预测了森林火灾的概率。。。
    使用python手动计算泊松分布？？？没有内置函数吗？？ 淦 还真没有 但是matlab有
    他在做什么：认为接下来十年内极端火灾情况的发生次数符合泊松分布，通过数据集获得了泊松分布的lambda，这时得到期望就可以了，但python没有直接给出泊松分布概率密度函数的方法，就为了可视化硬sample了一个。。。
    重来：每一个坐标点是一个样本，通过数据集获得了它发生极端火灾事件的次数，单位设置为10年，与题目相符，t代表考虑的10年，所以要除以t。本文想根据这些样本拟合泊松分布，现在有的是一个柱状图（离散的），能通过已有数据集计算出泊松分布的lambda，但是，python没有根据lambda直接计算，于是就自己设计了一个采样的方法画了出来，最后就是简单的算期望。。。其实期望完全可以不用画这个图啊。。。

  18. 极端情况怎么仿真：考虑变为过火面积的增加，输入原先的算法进行仿真，面积增加变为了1-10倍（最远着火点起，面积进行扩大）
    然后专门介绍了一下三次样条插值？？？不是很理解
    给结论+分析图线

  19. 第三个模型
    一个很巧妙的地方：前两问考虑平原地区也即是火灾最为频发的维多利亚东部，不去考虑山区的影响，最后一问加上山区影响，但不再从维多利亚东部考虑，这样不会否掉前面的模型（即 不用说前面的模型考虑不周）

  20. 禁忌搜索算法： 参考文章
    同样定义网格&邻居
    网络层？？？？？？
    （1）目的：启发式算法，用来摆脱局部最优
    搜索的过程中如果只搜索起始点附近邻域内的操作，易陷入局部最优！（爬山就是一个 搜索的邻域高度就是输出的特例）
    （2）禁忌搜索只设置了禁忌策略的搜索，对应的还有破禁准则，其实就和模拟退火一样自己定义了一套规则进行最优解的搜索
    属于是优启发式化算法之一

    文章怎么做的：即优化悬停位置，搜索悬停位置可能的解
    邻居定义为4邻居
    初始解：去一个邻居点，随后将它添加到禁忌表
    描述了一下算法的过程，这里只是描述性并没有把算法的每一个公式说出来（可以注意一下写作），仅有的一个公式将高度折合成了无线电的衰减距离，也不是算法的公式，但是至少应该引用一下吧，，，
    文中所谓的过拟合实际上就是没有跳出局部最优
    定义到达邻居的距离为邻居的级数，逐层搜索，每一层对禁忌表中的元素进行打破禁忌处理，比较逐层的最小值获得最最小值

2.2 ‘Rapid Bushfire Response’ UAV System

第二篇文章的图和公式数学含义不如第一篇了 嘤嘤嘤

  1. Abstract
    模型一：仍然区域离散的方法，但将每一个离散区域的信息划分为了不同的层（理解着是 三种信息的意思？）
    模型二：全球变暖情况下火灾的变化 - 类似养鱼那篇文章将温度映射到鱼群的前移，这篇文章将温度映射到了平均火灾概率再映射到了燃烧区域的大小（两篇文章都这样来理解严重火灾了！应该是查资料查到的！怎样映射：元胞自动机，这个太经典了。。。）
    模型三：考虑地形影响，相当于在模型一上的扩展 考虑通信：使用高度的二阶梯度信息
  2. 提模型阶段也是数据收集+数据筛选，也是独立于三个模型优先进行，他这个的database名字就规范多了。。。
  3. 还是做了热图以进行展示，还是建立了经纬度坐标之间的距离关系
